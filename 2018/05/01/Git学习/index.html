Git简介

什么是git

git是目前最流行的分布式版本控制工具，什么是版本控制呢，拿word来举例的话就是，“myword.docx”，“myword新.docx”、“myword最终版.docx”、“myword最终版打印.docx”、“myword最终版打印新.docx”……。利用git可以方便的管理这些版本，有版本修改说明，并且可以回溯到之前的版本，从而脱离手动版本控制。

版本 文件名 说明 时间
1 myword.docx xxx工作文档 2/1 21:12
2 myword.docx 增加了xxx 2/4 12:45
3 myword.docx 删除了xxx 2/5 12:11
4 myword.docx 修改了xxx 2/9 17:56

集中式与分布式

git采用的是分布式版本管理控制，那集中式和分布式有什么区别呢？

  • 集中式

    所谓集中式，就是版本库放在中央服务器，每次工作前要先从中央服务器拿过最新的版本，工作完毕后在将自己的工作推送到服务器。

  • 分布式

    分布式系统安全性要高很多，每个人的电脑里都有一个版本库，如果有一台电脑出故障，从其他人那里拷贝一份就好了，如果集中式系统的中央服务器瘫痪了，整个工作就gg了。git中也有一台电脑充当中央服务器，这是为了方便交换修改和合并工作，并不是必须的。git强大不止这样，还在于其强大的分支管理。

创建仓库

仓库就是版本库，英文名为repository，可以认为是一个目录，可以在里面添加、修改，也可以回溯到之前一个版本。

  1. 创建git仓库：git init

这样就创建好了一个仓库，并且可以看出创建了一个.git目录，这个目录是隐藏的，可以用ls -ah查看。

  1. 将文件添加到仓库：git add \

    简单起见，在根目录下（即工程目录，.git的父目录）创建一个gitstudy.txt，内容为“I’am studying git.”

并没有返回信息，没有返回信息就代表添加成功。

  1. 提交内容到版本库：git commit -m”说明”

    之前我们添加了一个文件，可以用git status查看当前状态

是说有文件需要提交（用git rm命令删除文件）

这段话是说1个文件被改动，插入了1行内容。

时光机穿梭

版本管理

  1. 版本信息查看：git log

    使用这个命令，可以查看仓库中有哪些版本

如果嫌输出信息太多，可以在命名最后添加–pretty=oneline参数

这一大串数字和字母便是版本号(commit id)，

  1. 回退到历史版本：git reset –hard 版本号

    首先要知道当前版本是那个，git log命令是按时间由近到远从上到下显示的，并且带有HEAD的便是当前版本。HEAD的上一个版本是HEAD^,上上一个是HEAD^^,也可以写成HEAD~1，HEAD~2。

可以写HEAD^1，也可以写版本号，版本号不用写全，写出前几位能供系统唯一区分出确定版本就行了。这时在看git log，发现没有了当前版本之后的记录，感觉像穿梭到古代回不去了呜呜呜……



$ git reflog

c223390 (HEAD -> master) HEAD@{0}: reset: moving to c2233
0dfc4c0 HEAD@{1}: reset: moving to 0dfc
c223390 (HEAD -> master) HEAD@{2}: reset: moving to c2233
0dfc4c0 HEAD@{3}: reset: moving to 0dfc
c223390 (HEAD -> master) HEAD@{4}: reset: moving to c2233
0dfc4c0 HEAD@{5}: commit: final
1b14239 HEAD@{6}: commit: null
9307367 HEAD@{7}: commit: null
c223390 (HEAD -> master) HEAD@{8}: commit (initial): first commit

这段话告诉了我们，公有部分是I’am studying git.。工作区比仓库最新版少了Git good one.，多了Git good one.Git good two.Git good three.。之所以少了Git good one.，又多了Git good one.，是因为我在这句话后面敲了回车。

要想将再次的修改提交到仓库中，就要在进行add和commit，也可以多次add后，在commit。
  1. 撤销修改

  • 没有add之前：git checkout – \

    在对工作区进行了修改，没有add之前，想撤销对工作区的修改，此时可以用git status 查看一下

这段高速我们用git add将会被用来commit，用git checkout可以撤销对工作区的修改。



git checkout后的--很重要，否则就会变成另一条命令。

  • add之后，没有commit之前：git reset HEAD

    此时，用git status查看一下状态

这段话告诉我们，用git reset HEAD \来清空缓存区。

这样就清空了缓存区，成了没有add之前的状态，之后再按add之前撤销对工作区的修改。
